1. Solution popisuje zakladne charakteristiky jazyka JAVA.
2. JAVA PLATFORM STANDARD EDITION (JavaEE) je VIRTUAL PLATFORM a TECHNICAL STANDARD na ktorom bezia JAVA PROGRAMS.
3. JAVA PLATFORM STANDARD EDITION obsahuje JAVA LANGUAGE SPECIFICATION a JAVA VIRTUAL MACHINE SPECIFICATION.
4. CODE v JAVE sa vyvija v JAVA DEVELOPMENT KIT (JDK). JAVA DEVELOPMENT KIT pozostava z nasledujucich casti.
   A. JAVA COMPILER.
   B. RUNTIME ENVIRONMENT, ktore sa nazyva JAVA VIRTUAL MACHINE (JVM).
5. JAVA DEVELOPMENT KIT je dostupnych na mnozstve PLATFORMS.
   A. WINDOWS.
   B. LINUX.
   C. MAC OS.
   D. SOLARIS.
6. JAVA je RELEASED kazdych 6 mesiacov, pricom kazde 2 roky je RELEASE LONG TERM SUPPORT RELEASE (LTS). Aktualna LTS je JAVA 25.
7. JAVA bola RELEASED prvykrat v roku 1995.
8. JAVA COMPILER kompiluje JAVA CODE do BYTECODE. Pre BYTECODE platia nasledujuce fakty.
   A. BYTECODE je virtualny BINARY CODE, ktory interpretuje JAVA VIRTUAL MACHINE.
   B. Pri spusteni JAVA PROGRAM je JAVA VIRTUAL MACHINE je zodpovedna za validaciu BYTECODE a jeho transformaciu do CODE pre dany PROCESSOR a OPERATING SYSTEM.
   C. JAVA VIRTUAL MACHINE pre JAVA PROGRAMS poskytuje rozlicne SERVICES ako je GARBAGE COLLECTOR, TYPE CHECKING konzistatneny MEMORY MODEL a THREADING MODEL.
9. JAVA je PLATFORM INDEPENDENT. PLATFORM INDEPENDENCE dosahuje nasledujucimi sposobmi.
   A. JAVA COMPILER generuje BYTE CODE, ktory je nezavisly na PROCESSOR PLATFORM a OPERATING SYSTEM.
   !!! B. JAVA definuje pre OPERATING SYSTEM RESOURCES (THREADS, DEVICES, GUI) ABSTRACTIONS voci ktorym su vytvarane JAVA PROGRAMS. JAVA VIRTUAL MACHINE vykonava konverziu tychto ABSTRACTIONS na realne RESOURCES daneho OPERATING SYSTEM.
10. JAVA predstavuje VIRTUAL JAVA PLATFORM, ktora je nezavislou PLATFORM, ktoru je mozne vdaka JAVA VIRTUAL MACHINE implementovat nad roznymi PROCESSOR PLATFORMS a OPERATING SYSTEMS. JAVA PLATFORM sa sklada z nasledujucich COMPONENTS.
   A. JAVA PROGRAMMING LANGUAGE.
   B. JAVA VIRTUAL MACHINE.
   C. JAVA STANDARD LIBRARIES.
11. Pre JAVA STANDARD LIBRARIES platia nasledujuce fakty.
   A. JAVA STANDARD LIBRARIES poskytuju zakladne funkcionality ako pracu s DATA STRUCTURES, STRING PROCESSING, DATE/TIME PROCESSING, GUI, INPUT/OUTPUT OPERATIONS, THREADING, NETWORK OPERATIONS.
   B. JAVA STANDARD LIBRARIES sluzia ako zaklad pre implementaciu sofistikovanejsich LIBRARIES, ako napriklad DATABASE LIBRARIES ci WEB SERVICE LIBRARIES.
!!! 12. JAVA PLATFORM je OBJECT ORIENTED, ale PRIMITIVE TYPES nie su reprezentovane ako OBJECT. To je rozdiel oproti .NET, kde vsetky TYPPES, aj PRIMITIVE TYPES mozu byt BOXED do OBJECTS.
13. JAVA implementuje JUST IN TIME COMPILATION. Pre JUST IN TIME COMPILATION platia nasledujuce fakty.
   A. JUST IN TIME COMPILATION vykonava COMPILATION BYTECODE do MACHINE CODE.
   B. JUST IN TIME COMPILATION umoznuje vykonavat CODE OPTIMALIZATIONS, pretoze ma znalost o PROCESSOR PLATFORM a tym padom moze pri OPTIMALIZATION moze vykonavat OPTIMALIZATION pre konkretne PROCESSOR INSTRUCTIONS.
   !!! C. JUST IN TIME COMPILATION nemusi byt vzdy aplikovana, ak JAVA VIRTUAL MACHINE zisti, ze by generovany CODE neviedol k vyssej PERFORMANCE. V takom pripade JAVA vykonava INTERPRETATION BYTE CODE, namiesto vykonania JUST IN TIME COMPILATION.
   D. JUST IN TIME COMPILATION monitoruje vykonnost COMPILED CODE, a moze vykonat RECOMPILATION, ak zisti, ze v kontexte v ktorom je CODE pouzivany moze byt po RECOMPILATION vykonany rychlejsie.
14. JAVA pouziva REFERENCES a NEPOUZIVA POINTERS.
15. JAVA implementuje GARBAGE COLLECTOR. Pre GARBAGE COLLECTOR platia nasledujuce fakty.
   A. GARBAGE COLLECTOR odstranuje OBJECTS, na ktore neexistuju REFERNCES v PROGRAME.
   B. GARBAGE COLLECTOR bezi v separatnom THREAD.
   C. GARBAGE COLLECTOR moze bezat na viacerych THREADS.
16. JAVA implementuje ERROR HANDLING pomocou EXCEPTIONS.
17. JAVA nepodporuje OPERATOR OVERLOADING.
18. JAVA nepodporuje PREPROCESSORS.
19. JAVA PLATFORM je OPEN SOURCE. To znamena, ze OPEN SOURCE je JAVA COMPILER, JAVA VIRTUAL MACHINE i JAVA STANDARD LIBRARIES.
20. JAVA, na rozdiel od .NET existuje v 2 verziach.
   A. JAVA PLATFORM STANDARD EDITION (JavaSE) je urcena pre vseobecne PROGRAMY vytvorene v JAVE.
   B. JAKARTA ENTERPRISE EDITION (JavaEE) je EXTENSION pre ENTERPRISE SYSTEMS.
21. ORACLE definuje viacero nezavislych JAVA PLATFORM STANDARDS, ktore umoznuju implementaciu JAVA PLATFORMS, ktore mozu bezat na rozlicnych typoch DEVICES. ORACLE definuje nasledujuce JAVA PLATFORM STANDARDS.
   A. JAVA PLATFORM STANDARD EDITION. Pre JAVA PLATFORM STANDARD EDITION platia nasledujuce fakty.
      A. JAVA PLATFORM STANDARD EDITION, ktora umoznuje beh JAVA PROGRAMS na standardnych DEVICES ako su DESKTOP COMPUTERS, NOTEBOOKS ci SERVERS.
      B. JAVA PLATFORM STANDARD EDITION obsahuje vsetku funkcionalitu potrebnu pre vyvoj JAVA PROGRAMS.
      !!!!! C. JAVA PLATFORM STANDARD EDITION je JAVA PLAFORM STANDARD, ktory ma viacero nezavislych IMPLEMENTATIONS, ako je napriklad OPEN JDK.
      D. JAVA PLATFORM STANDARD EDITION ma vysoku mieru spatnej kompatibility. To znamena, ze JAVA PROGRAMS napisane na predosle VERSIONS, az na par vynimiek, vzdy su spustitelne na vyssich verziach JAVA PLATFORM STANDARD EDITION.
      E. JAVA PLATFORM STANDARD EDITION je RELEASED kazdych 6 mesiacov.
      F. Aktualna verzia JAVA PLATFORM STANDARD EDITION je JAVA 25.
   B. JAVA PLATFORM MICRO EDITION. Pre JAVA PLATFORM MICRO EDITION platia nasledujuce fakty.
      A. JAVA PLATFORM MICRO EDITION je urcena pre beh na DEVICES, ktore maju menej ako 1 MB MEMORY.
      B. JAVA PLATFORM MICRO EDITION je zakladom pre JAVA TV RUNTIME ENVIRONMENT pouzivany v SMART TV.
      C. JAVA PLATFORM MICRO EDITION bola do velkej miery nahradena ANDROIDOM, ktory je postaveny na JAVE.
   C. JAVA CARD. Pre JAVA CARD platia nasledujuce fakty.
      A. JAVA CARD je JAVA PLATFORM STANDARD umoznujuci bez JAVA PROGRAMS na SMART CARDS, ako su napriklad SIM CARDS.
      B. V JAVA CARD JAVA COMPILER generuje BYTECODE, ktory je podobny, no nie identicky voci standardnemu BYTECODE.
      C. V JAVA CARD je BYTECODE vykonavany JAVA CARD VIRTUAL MACHINE, ktora bezi v SMART CARD.
      D. JAVA CARD VIRTUAL MACHINE nie je kompatibilna so standardnym JAVA VIRTUAL MACHINE pre obmedzenia SMART CARDS.
      E. JAVA CARD VIRTUAL MACHINE nepodporuje THREADING, ani GARBAGE COLLECTOR.
      F. JAVA CARD VIRTUAL MACHINE ma limitovany LIBRARY, ktora neobsahuje niektora CLASSES, ale obsahuje silne kryptograficke algoritmy, ako aj LIBRARY, ktora umoznuje komunikaciu SMART CARDS s inymi DEVICES.
      G. JAVA CARD existuje vo verzii CLASSIC EDITON a CONNECTED EDITION. CONNECTED EDITION podporuje THREADING a GARBAGE COLLECTOR.
   D. JAKARTA ENTERPRISE EDITION. Pre JAKARTA ENTERPRISE EDITION platia nasledujuce fakty.
      A. JAKARTA ENTERPRISE EDITION je rozsirenim JAVA PLATFORM STANDARD EDITION pre implementaciu ENTERPRISE PROGRAMS v JAVE.
      B. JAKARTA ENTERPRISE EDITION obsahuje mnozstvo LIBRARY SPECIFICATIONS ako napriklad ENTERPRISE JAVA BEANS, CDI pre DEPENDENCI INJECTION, SERVLETS, JSP, JFS ci JAVA MAIL API.
      !!!!! C. JAKARTA ENTERPRISE EDITION je JAVA PLAFORM STANDARD, ktory ma viacero nezavislych IMPLEMENTATIONS, ako je napriklad ECLIPSE GLASS FISH.
      !!!!! D. JAVA SPRING BOOT je alternativna LIBRARY, ktora definuje podobnu funkcionalitu ako JAKARTA ENTERPRISE EDITION, ale je vyvijana nezavislym timom DEVELOPERS a je konkurenciou voci JAKARTA ENTERPRISE EDITION.
22. REAL TIME JAVA (RT-JAVA) je JAVA PLATFORM STANDARD umoznujuci beh JAVA PROGRAMS nad REAL TIME OPERATING SYSTEMS. Tato PLAFORM je vsak silne zviazana so samotnym REAL TIME OPERATING SYSTEMS.
23. JAVA PLATFORM STANDARD EDITION ma viacero nezavislych IMPLEMENTATIONS. Pre JAVA PLATFORM STANDARD EDITION IMPLEMENTATIONS platia nasledujuce fakty.
   A. JAVA PLATFORM STANDARD EDITION je len JAVA PLATFORM STANDARD na zaklade ktoreho su implementovane JAVA PLATFORM STANDARD EDITION IMPLEMENTATIONS.
   B. JAVA PLATFORM STANDARD EDITION IMPLEMENTATION obsahuje implementaciu JAVA COMPILER, JAVA VIRTUAL MACHINE a JAVA LIBRARIES.
   C. JAVA PLATFORM STANDARD EDITION IMPLEMENTATION musi prejst TECHNOLOGY COMPATIBILITY KIT TESTS, ktore skontroluju funkcnost JAVA PLATFORM STANDARD EDITION IMPLEMENTATION. Ak su tieto TESTS uspesne vykonane, JAVA PLATFORM STANDARD EDITION IMPLEMENTATION ziska JAVA SE COMPATIBLE CERTIFICATION.
   D. JAVA PLATFORM STANDARD EDITION ma nasledujuce IMPLEMENTATIONS.
      A. OPEN JDK. Pre OPEN JDK platia nasledujuce fakty.
         !!! A. OPEN JDK nie je oficialna implementacia JAVA PLATFORM STANDARD EDITION IMPLEMENTATION. To je ORCALE JDK. Ale OPEN JDK prebrala mnohe technologie z ORCALE JDK.
         B. OPEN JDK je licencovana podla GPL licencie.
         C. OPEN JDK je BUILT viacerymi PROVIDERS, ako ECLIPSE, RED HAT, SAP, AMAZON ci APPLE.
      B. ORCALE JDK. Pre ORCALE JDK platia nasledujuce fakty.
         !!! A. ORCALE JDK nie je oficialna implementacia JAVA PLATFORM STANDARD EDITION IMPLEMENTATION.
         B. ORCALE JDK je licencovana podla ORACLE NO-FEE licencie.
         C. ORCALE JDK je RELEASED kazdych 6 mesiacov, pricom kazde 2 roky je vydana LONG TERM SUPPORT verzia ORCALE JDK, ktora je podporovana 3 roky.
24. JAVA moze byt instalovana zo WEBU ORACLE nasledujucimi sposobmi.
   A. Pomocou EXE FILE.
   B. Pomocou MSI FILE.
   C. Pomocou ZIP FILE. ZIP FILE vyzaduje manualnu registraciu PATH k JAVA COMPILER a JAVA INTERPRETER v ENVIRONMENT VARIABLE [PATH].
25. JAVA INSTALLATION obsahuje nasledujuce DIRECTORIES.
   A. DIRECTORY [bin]. DIRECTORY [bin] obsahuje JAVA TOOLS vratane JAVA INTERPRETER [java] a JAVA COMPILER [javac].
   B. DIRECTORY [conf]. DIRECTORY [conf] obsahuje JAVA CONFIGURATION FILES.
   C. DIRECTORY [include]. DIRECTORY [include] obsahuje C HEADER FILES pre BINDING JAVA do C PROGRAMS.
   D. DIRECTORY [jmods]. DIRECTORY [jmods] obsahuje JAVA MODULES z JDK, ako napriklad BASE MODULE.
   E. DIRECTORY [legal]. DIRECTORY [legal] obsahuje pravne dokumenty pre pouzivanie JAVY.
   F. DIRECTORY [lib]. DIRECTORY [lib] obsahuje interne JAVA TOOLS. FILE [lib/src.zip] obsahuje SOURCE CODES PUBLIC LIBRARIES.
   G. DIRECTORY [release]. DIRECTORY [release] obsahuje FILE s KEY VALUE PAIRS.
26. Po instalacii JAVA JDK, je mozne overit jej funkcnost pomocou COMMAND [javac -version].
!!! 27. V pripade, ze JAVA INSTALLATION je vykonavany zo ZIP FILE, je nutne po nainstalovani vlozit do ENVIRONMENT VARIABLE [PATH] PATH na DIRECTORY [bin].
28. JAVA FILE je COMPILED pomocou COMMAND [javac FILENAME], kde VALUE [FILENAME] obsahuje PATH k JAVA FILE. Vysledkom COMPILATION je CLASS FILE s EXTENSION [.class].
!!! 29. JAVA FILE sa spusta pomocou COMMAND [java FILENAME], kde VALUE [FILENAME] NESMIE obsahovat EXTENSION [.class].
!!! 30. JAVA umoznuje SINGLE FILE PROGRAMS spustat pomocou COMMAND [java FILENAME], kde VALUE [FILENAME] MUSI obsahovat EXTENSION [.java].
31. JAVA PROJECTS mozu byt vytvarana nasledujucimi sposobmi.
   A. SIMPLE JAVA PROJECTS. SIMPLE JAVA PROJECTS su priamo COMPILED pomocou COMMAND LINE TOOL [javac].
   B. MAVEN JAVA PROJECTS. MAVEN JAVA PROJECTS sluzia pre vytvaranie velkych PROJECTS, ktore vyuzivaju ine LIBRARIES.
32. MAVEN je BUILD TOOL umoznujuci BUILD velkych JAVA PROJECTS. Pre MAVEN platia nasledujuce fakty.
   A. MAVEN je je BUILD TOOL umoznujuci BUILD velkych JAVA PROJECTS.
   B. MAVEN vykonava BUILD celeho JAVA PROJECT, takze na COMPILATION nie je nutne pouzit COMMAND LINE TOOL [javac].
   C. MAVEN definuje obsah JAVA PROJECT vo FILE [pom.xml].
   D. FILE [pom.xml] obsahuje kompletnu konfiguraciu JAVA PROJECT, vratane vsetkych DEPENDENCIES na EXTERNAL LIBRARIES, ktore JAVA PROJECT vyuziva.
   E. FILE [pom.xml] je ulozeny v ROOT DIRECTORY MAVEN PROJECT.
   F. FILE [pom.xml] obsahuje PROJECT PROPERTIES, nazyvane COORDINATES. FILE [pom.xml] obsahuje nasledujuce COORDINATES.
      A. VALUE [GROUP ID] reprezentuje PACKAGE NAME. Reprezentuje COMPANY, ktory JAVA PROJECT vytvorila.
      B. VALUE [ARTIFACT ID] reprezentuje ARTIFACT NAME, co je nazov samotneho JAVA PROJECT.
      C. VALUE [VERSION] obsahuje VERSION daneho JAVA PROJECT.
33. Aj ked JAVA PROJECTS mozu pouzivat lubovolnu DIRECTORY STRUCUTRE, podla CONVENTION, ktoru pouziva MAVEN sa odporuca nasledujuce DIRECTORY STRUCTURE JAVA PROJECT.
   A. DIRECTORY [src/main/java]. DIRECTORY [src/main/java] obsahuje SOURCE CODES JAVA PROJECT.
   B. DIRECTORY [src/main/resources]. DIRECTORY [src/main/resources] obsahuje RESOURCES JAVA PROJECT.
   C. DIRECTORY [src/test/java]. DIRECTORY [src/test/java] obsahuje TEST CLASSES JAVA PROJECT.
   D. DIRECTORY [src/test/resources]. DIRECTORY [src/test/resources] obsahuje RESOURCES pre TEST CLASSES JAVA PROJECT.
   E. DIRECTORY [src/site]. DIRECTORY [src/site] obsahuje WEB SITE JAVA PROJECT.
   F. DIRECTORY [target]. DIRECTORY [target] obsahuje COMPILED FILES JAVA PROJECT.
   G. FILE [LICENSE.txt]. FILE [LICENSE.txt] obsahuje licencne podmienky JAVA PROJECT.
   H. FILE [NOTICE.txt]. FILE [NOTICE.txt] obsahuje DEPENDENCIES JAVA PROJECT na EXTERNAL LIBRARIES.
   I. FILE [README.txt]. FILE [README.txt] obsahuje README FILE JAVA PROJECT.
34. JAVA PROJECTS je mozne vyvijat v nasledujucich DEVELOPMENT ENVIRONMENTS.
   A. ENVIRONMENT [IntelliJ IDEA].
   B. ENVIRONMENT [Eclipse IDE]. ECLIPSE existuje v nasledujucich verziach.
      A. VERSION [Eclipse IDE for Java Developers]. Obsahuje zakladne TOOLS pre vyvoja JAVA PROGRAMS.
      B. VERSION [Eclipse IDE for Enterprise Java and Web Developers]. Obsahuje pokrocile TOOLS pre vyvoja JAVA PROGRAMS.
   C. ENVIRONMENT [VS Code].
   D. ENVIRONMENT [NetBeans].
