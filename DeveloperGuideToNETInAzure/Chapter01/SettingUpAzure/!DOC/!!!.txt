1. Solution demonstruje registraciu AZURE SUBSCRIPTION a konfiguraciu DEVELOMENT ENVIRONMENT, vratane VISUAL STUDIO.
2. AZURE SUBSCRIPTION sa vytvara na URL [https://azure.microsoft.com/en-in/free/].
3. RESOURCES v AZURE mozu byt vytvarane nasledujucimi sposobmi.
   A. Pomocou AZURE UI.
   B. Pomocou COMMAND LINE TOOLS ako TOOL [AZURE CLI] a TOOL [AZURE POWER SHELL].
4. RESOURCES v AZURE mozu byt zaradzovane do RESOURCE GROUP. Pre RESOURCE GROUP platia nasledujuce fakty.
   A. RESOURCE GROUPS sluzia na zaradzovanie pribuznych RESOURCES do toho isteho RESOURCE GROUP.
   B. RESOURCE GROUP je RESOURCE v AZURE, ktory sa vytvara v AZURE podobne ako ine RESOURCES.
5. RESOURCES v AZURE mozu mat priradene TAGS. Pre TAGS platia nasledujuce fakty.
   A. TAGS su METADATA asociovane s danym RESOURCES.
   B. TAG je KEY VALUE PAIR, ktory moze obsahovat lubovolne hodnoty.
   C. TAGS sluzia na kategorizaciu RESOURCES v AZURE. Na ich zaklade je mozne RESOURCES v AZURE vyhladavat.
6. AZURE DEVELOPMENT vyzaduje instalaciu .NET SDK prislusnej verzie .NET.
7. VISUAL STUDIO pre AZURE DEVELOPMENT MUSI mat nainstalovany WORKLOAD [AZURE DEVELOPMENT] pomocou VISUAL STUDIO INSTALLER.
!!!!! 8. ACCOUNT, ktory bol pouzity na vytvorenie AZURE SUBSCRIPTION MUSI byt pouzity aj vo VISUAL STUDIO ako VISUAL STUDIO ACCOUNT.
