1. Solution demonstruje pouzitie AZURE SQL DATABASE.
2. AZURE podporuje SQL SERVER DATABASES ako nasledujuce SERVICES.
   A. SQL SERVER ON AZURE VIRTUAL MACHINES.
   B. AZURE SQL MANAGED INSTANCE.
   C. AZURE SQL DATABASE.
3. Pre SQL SERVER ON AZURE VIRTUAL MACHINES platia nasledujuce fakty.
   A. SERVICE predstavuje INFRASTRUCTURE AS A SERVICE (IAAS).
   B. SQL SERVER bezi v dedikovanej AZURE VIRTUAL MACHINE.
   C. SQL SERVER beziaci vo VIRTUAL MACHINE zabezpecuje 100% kompatibilitu s ON PREMISE SQL SERVERS.
   D. SERVICE zabezpecuje pristup k VIRTUAL MACHINE na urovni OPERATING SYSTEM ci umoznuje kompletny manazment OPERATING SYSTEM a SQL SERVER.
   E. SQL SERVER moze bezat na WINDOWS, alebo LINUX.
   F. SQL SERVER moze bezat pod FREE LICENSE, alebo pod DEDICATED LICENSE.
   !!! G. USER je plne zodpovedny za manazment SQL SERVER, vratane BACKUP MANAGEMENT ci AVAILABILITY.
   H. SERVICE je vhodny pre jednoduchu migraciu existujucich SQL SERVER PROGRAMOV do AZURE.
4. Pre AZURE SQL MANAGED INSTANCE platia nasledujuce fakty.
   A. SERVICE predstavuje PLATFORM AS A SERVICE (PAAS).
   B. SERVICE podporuje takmer vsetky INSTANCE LEVEL a DATABASE LEVEL CAPABILITIES, ale uz nie manazment na urovni OPERATING SYSTEM.
   C. Na rozdiel od SQL SERVER ON AZURE VIRTUAL MACHINES, SERVICE poskytuje BUILT IN SERVICES ako PATCHING, BACKUP MANAGEMENT a RECOVERY MANAGEMENT.
   D. SERVICE je vhodny pre jednoduchu migraciu existujucich SQL SERVER PROGRAMOV do AZURE.
5. Pre AZURE SQL DATABASE platia nasledujuce fakty.
   A. SERVICE poskytuje SQL SERVER, ktory je spravovany AZURE PLATROFM.
   B. SERVICE NEPODPORUJE vsetky funkcionality TRANSACT SQL a preto migracia existujucich PROGRAMOV na tento SERVICE je narocnejsia.
   C. SERVICE umoznuje beh SQL SERVERS na dedikovanych VIRTUAL MACHINES, alebo POOL SHARED VIRTUAL MACHINES.
6. AZURE SQL DATABASE podporuje nasledujuce typy DEPLOYMENTS.
   A. SINGLE DATABASE DEPLOYMENT.
   B. ELASTIC POOL DEPLOYMENT.
7. Pre AZURE SQL DATABASE SINGLE DATABASE DEPLOYMENT platia nasledujuce fakty.
   A. DATABASE bezi na dedikovanej instancie SQL SERVER.
   B. SQL SERVER ma pridelene vlastne dedikovane RESOURCES.
   C. SINGLE DATABASE DEPLOYMENT podporuje nasledujuce typy TIERS.
      A. VIRTUAL CORE BASED PURCHASING MODEL. Pre tento PURCHASING MODEL platia nasledujuce fakty.
         A. DATABASE bezi v SERVERLESS MODE, kde pridelovanie DATABASE jednotlivych VIRTUAL CORES je dynamocne a automaticky skalovane podla vyuzivanie DATABASE.
         !!! B. Pridelovanie VIRTUAL CORES je plne automaticky na zaklade zataze. V pripade, ze DATABASE je urcity cas nepozivana, tak su jej odobrate VIRTUAL CORES.
         C. USER plati pocet sekund, pocas ktorych realne bezi DATABASE na VIRTUAL CORES.
         D. PURCHASING MODEL je vhodny pre PROGRAMS, ktore maju velku variablitu pouzitia, ktora sa v case meni. AUTOSCALING umoznuje efektivne pridelovanie VIRTUAL CORES beziadim DATABASES.
         !!!!! E. PROGRAMS vyuzivajuce tento PURCHASING MODEL musia implementovat RETRY logiku pri pristupe k DATABASE. Je to preto, lebo nevyuzivanym DATABASES su odoberane VIRTUAL CORES a tym padom nebeziacim DATABASES moze nejaky cas trvat kym nabehnu.
      B. DATABASE TRANSACTION UNIT BASED PURCHASING MODEL. Pre tento PURCHASING MODEL platia nasledujuce fakty.
         A. DATABASE TRANSACTION UNIT BASED PURCHASING MODEL ma predefinovany pocet VIRTUAL CORES, ktore su vzdy spustene.
         B. USER plati za pocet hodin pocas ktorych VIRTUAL CORES bezia.
8. Pre AZURE SQL DATABASE ELASTIC POOL DEPLOYMENT platia nasledujuce fakty.
   A. DATABASES zidelaju DATABASE RESOURCES, ktore su automaticky pridelovane z DATABASE RESOURCE POOL.
   B. ELASTIC POOL DEPLOYMENT poskytuje efektivny SCALING, kde RESOURCES su DATABASES pridelovane, iba ak su realne vyuzivane.
   C. DATABASES mozu byt dynamicky pridavane a odoberane z ELASTIC POOL.
9. Vytvaranie a odstranovanie instancii SQL SERVERS sa robi pomocou nasledujucich COMMANDS.
   A. Nova instancia SQL SERVER sa vytvara pomocou COMMAND [az sql server create].
   B. Existujuca instancia SQL SERVER sa odstranuje pomocou COMMAND [az sql server delete].
   C. Zoznam existujucich instancii SQL SERVER sa vracia pomocou COMMAND [az sql server list].
10. Vytvaranie a odstranovanie DATABASES sa robi pomocou nasledujucich COMMANDS.
   A. Nova DATABASE sa vytvara pomocou COMMAND [az sql db create].
   B. Existujuca DATABASE sa odstranuje pomocou COMMAND [az sql db delete].
   C. Zoznam existujucich DATABASES beziacich instancii SQL SERVER sa vracia pomocou COMMAND [az sql db list].
11. AZURE RESOURCE MANAGER TEMPLATES (ARM TEMPLATES) umoznuju DEPLOY roznych AZURE SERVICES pomocou JSON FILES. Pre AZURE RESOURCE MANAGER TEMPLATES platia nasledujuce fakty.
   A. AZURE RESOURCE MANAGER TEMPLATES predstavuje mechanizmus INFRASTRUCTURE AS CODE (IaC), ktory umoznuje PROGRAMOVO definovat pozadovane AZURE RESOURCES vo forme JSON FILES, ktore je potom mozne vlozit do AZURE.
   B. AZURE RESOURCE MANAGER TEMPLATES pouzivaju JSON FILES na popis RESOURCES, ktore maju byt v AZURE vytvorene.
   C. AZURE RESOURCE MANAGER TEMPLATES umoznuju pomocou JSON FILES definovat RESOURCES, ako aj ich CONFIGURATION SETTINGS, ktore maju pri vytvarani tychto RESOURCES sa aplikovat.
   D. AZURE RESOURCE MANAGER TEMPLATES JSON FILES mozno importovat do AZURE pomocou CLI COMMANDS.
   E. AZURE RESOURCE MANAGER TEMPLATES JSON FILES mozno exportovat z AZURE pomocou AZURE PORTAL. Exportovany FILE [.ZIP] obsahuje nasledujuce FILES.
      A. FILE [template.json] obsahuje informacie o RESOURCE, ktopry je pomocou AZURE RESOURCE MANAGER TEMPLATE vytvarany.
      B. FILE [parameters.json] obsahuje PARAMETERS daneho RESOURCE, ktore sa nastavia pri jeho vytvarani.
   !!! F. Alternativou k AZURE RESOURCE MANAGER TEMPLATES je AZURE BICEP, co je PROGRAMMING LANGAUGE, ktory umoznuje definovat RESOURCES pre AZURE RESOURCE MANAGER, podobne ako JSON FILES v AZURE RESOURCE MANAGER TEMPLATES.
12. RESOURCES sa pomocou AZURE RESOURCE MANAGER TEMPLATES odstranuju pomocou nasledujucich COMMANDS.
   A. Novy RESOURCE DEPLOYMENT sa vytvara pomocou COMMAND [az deployment group create]. COMMAND zaroven vytvara RESOURCES, ktore RESOURCE DEPLOYMENT obsahuje.
   B. Existujuci RESOURCE DEPLOYMENT sa odstranuje pomocou COMMAND [az deployment group delete].
   C. Zoznam existujucich RESOURCE DEPLOYMENTS sa vracia pomocou COMMAND [az deployment group list].
13. K AZURE SQL DATABASE sa je mozne pripojit pomocou ENTITY FRAMEWORK CORE. Platia nasledujuce fakty.
   A. ENTITY FRAMEWORK CORE musi pouzit standardny SQL SERVER PROVIDER na pristup k AZURE SQL DATABASE. PROVIDER je ulozeny v PACKAGE [Microsoft.EntityFrameworkCore.SqlServer].
   B. ENTITY FRAMEWORK CORE vyzaduje CONNECTION STRING. CONNECTION STRING sa da ziskat z AZURE PORTAL v sekcii [MY_DATABASE->Overview->Connection strings], alebo pomocou COMMAND [az sql db show-connection-string]. Prikladom je COMMAND [az sql db show-connection-string --server ta76dbserver --name MyDatabase --client "ado.net" --auth-type "ADIntegrated"].
14. Pripojenie sa k AZURE SQL DATABASE vyzauje nasledujuce konfiguracie.
   A. Vytvorit SQL SERVER pomocou COMMAND [az sql server create]. Prikladom je COMMAND [az sql server create --name ta76dbserver --resource-group Chapter07 --location westus --admin-user mydbadmin --admin-password 'Qaxwsx123+.'].
   B. Vytvorit SQL SERVER DATABASE pomocou COMMAND [az sql db create]. Prikladom je COMMAND [az sql db create --name MyDatabase --server ta76dbserver --resource-group Chapter07 --compute-model Provisioned --edition Basic].
   C. Povolit IP ADDRESS LOCAL MACHINE na pristup k DATABASE pomocou COMMAND [az sql server firewall-rule create]. Prikladom je COMMAND [az sql server firewall-rule create --name rulemyip --resource-group Chapter07 --server ta76dbserver --start-ip-address 178.40.58.85 --end-ip-address 178.40.58.85].
   D. Povolit AZURE ACTIVE DIRECTORY ACCOUNT pristup k SQL SERVER pomocou COMMAND [az sql server ad-admin create]. Prikladom je COMMAND [az sql server ad-admin create --resource-group Chapter07 --server ta76dbserver --object-id "5138f3d7-3ec0-4d89-97f5-40a6f6000528" --display-name "Timmy Anderson"].
   E. Povolit AZURE ACTIVE DIRECTORY ACCESS ONLY pristup k SQL SERVER pomocou COMMAND [az sql server ad-only-auth enable]. Prikladom je COMMAND [az sql server ad-only-auth enable --resource-group Chapter07 --name ta76dbserver].
   !!!!! F. Ak maju k SQL SERVER pristupovat ine AZURE SERVICE ako napriklad WEB APPLICATIONS, je nutne povolit pristup na FIREWALL pomocou COMMAND [az sql server firewall-rule create --name AllowAzureService --resource-group Chapter07 --server ta76dbserver --start-ip-address 0.0.0.0 --end-ip-address 0.0.0.0].
15. S AZURE SQL SERVER FIREWALL RULES sa pracuje pomocou nasledujucich COMMNDS.
   A. Novy AZURE SQL SERVER FIREWALL RULE sa vytvara pomocou COMMAND [az sql server firewall-rule create]. Prikladom je COMMAND [az sql server firewall-rule create --name rulemyip --resource-group Chapter07 --server ta76dbserver --start-ip-address 178.40.58.85 --end-ip-address 178.40.58.85].
   B. Existujuci AZURE SQL SERVER FIREWALL RULE sa odstranuje pomocou COMMAND [az sql server firewall-rule delete]. Prikladom je COMMAND [az sql server firewall-rule delete --name rulemyip --resource-group Chapter07 --server ta76dbserver].
   C. Zoznam AZURE SQL SERVER FIREWALL RULES vracia COMMAND [az sql server firewall-rule list]. Prikladom je COMMAND [az sql server firewall-rule list --resource-group Chapter07 --server ta76dbserver].
16. S AZURE SQL SERVER ACTIVE DIRECTORY ADMINISTRATOR ACCOUNTS sa pracuje pomocou nasledujucich COMMNDS.
   A. Novy AZURE SQL SERVER ACTIVE DIRECTORY ADMINISTRATOR ACCOUNT sa registruje pomocou COMMAND [az sql server ad-admin create]. Prikladom je COMMAND [az sql server ad-admin create --resource-group Chapter07 --server ta76dbserver --object-id "5138f3d7-3ec0-4d89-97f5-40a6f6000528" --display-name "Timmy Anderson"].
   B. Existujuci AZURE SQL SERVER ACTIVE DIRECTORY ADMINISTRATOR ACCOUNT sa registruje pomocou COMMAND [az sql server ad-admin delete --resource-group Chapter07 --server ta76dbserver]. Prikladom je COMMAND [az sql server ad-admin delete --resource-group Chapter07 --server ta76dbserver].
   C. Zoznam AZURE SQL SERVER ACTIVE DIRECTORY ADMINISTRATOR ACCOUNTS vracia COMMAND [az sql server ad-admin list]. Prikladom je COMMAND [az sql server ad-admin list --resource-group Chapter07 --server ta76dbserver].
17. S AZURE SQL SERVER ACTIVE DIRECTORY ONLY SETTINGS sa pracuje pomocou nasledujucich COMMNDS.
   A. AZURE SQL SERVER ACTIVE DIRECTORY ONLY SETTINGS sa povoluje pomocou COMMAND [az sql server ad-only-auth enable]. Prikladom je COMMAND [az sql server ad-only-auth enable --resource-group Chapter07 --name ta76dbserver].
   B. AZURE SQL SERVER ACTIVE DIRECTORY ONLY SETTINGS sa blokuje pomocou COMMAND [az sql server ad-only-auth disable]. Prikladom je COMMAND [az sql server ad-only-auth disable --resource-group Chapter07 --name ta76dbserver].
   C. Aktualny AZURE SQL SERVER ACTIVE DIRECTORY ONLY SETTINGS vracia COMMAND [az sql server ad-only-auth get]. Prikladom je COMMAND [az sql server ad-only-auth get --resource-group Chapter07 --name ta76dbserver].
18. MIGRATIONS sa aplikuju pomocou COMMAND [dotnet ef database update InitialMigration].
19. Pre AUTHENTICATION v AZURE SQL SERVER platia nasledujuce fakty.
   A. AZURE SQL SERVER podporuje viacere sposoby AUTHENTICATION, od USER/PASSWORD AUTHENTICATION, az po MANAGED IDENTITY AUTHENTICATION.
   B. AUTHENTICATION sa vykonava na nasledujucich urovniach.
      A. Na urovni AZURE SQL SERVER.
      B. Na urovni AZURE SQL SERVER DATABASE.
   C. AUTHENTICATION na urovni AZURE SQL SERVER DATABASE je vykonavana pomocou CONNECTION STRING, ktory musi obsahovat vsetky potrebne informacie pre vykonanie AUTHENTICATION.
   !!!!! D. Mnohe typy AUTHENTICATION, ako napriklad MANAGED IDENTITIES funguju BY-DEFAULT IBA pre v AZURE ENVIRONMENT. To znamena, ze AUTHENTICATION mozu pouzivat iba AZURE SERVICES ako napriklad WEB APPLICATION, ale nemozu ich pouzivat CLIENT PROGRAMS, ktore su spustene v externom prostredi.
   E. AZURE SQL SERVER podporuje nasledujuce typy AUTHENTICATIONS.
      A. AUTHENTICATION [ACTIVE DIRECTORY DEFAULT].
      B. AUTHENTICATION [SQL USER NAME PASSWORD].
      C. AUTHENTICATION [SYSTEM ASSIGNED MANAGED IDENTITY].
      D. AUTHENTICATION [USER ASSIGNED MANAGED IDENTITY].
20. Pre AUTHENTICATION [ACTIVE DIRECTORY DEFAULT] platia nasledujuce fakty.
   A. Tento typ AUTHENTICATION je mozne pouzit aj z EXTERNAL ENVIRONMENTS, ktore bezia mimo AZURE.
   B. AUTHENTICATION ziskava CREDENTIALS s rozlicnych zdrojov, ako napriklad ENVIRONMENT VARIABLES, AZURE CLI STORAGE, AZURE POWER SHELL STORAGE, VISUAL STUDIO STORAGE.
   C. AUTHENTICATION vyzaduje, aby CREDENTIALS boli zapisane v niektorom zo STORAGES z ktorych sa ich pokusa ziskat.
   D. AUTHENTICATION sa vykonava v nasledujucich krokoch.
      A. Vytvori sa AZURE SQL SERVER pomocou COMMAND [az sql server create]. Prikladom je COMMAND [az sql server create --name ta76dbserver --resource-group Chapter07 --location westus --admin-user mydbadmin --admin-password "Qaxwsx123+."].
      B. Na AZURE SQL SERVER sa vytvori FIREWALL RULE, ktory povoli pristup z LOCAL MACHINE pomocou COMMAND [az sql server firewall-rule create]. Prikladom je COMMAND [az sql server firewall-rule create --name rulemyip --resource-group Chapter07 --server ta76dbserver --start-ip-address 178.40.58.85 --end-ip-address 178.40.58.85].
      C. AZURE USER ACCOUNT sa nastavi ako ADMINISTRATOR ACCOUNT pre AZURE SQL SERVER pomocou pomocou COMMAND [az sql server ad-admin create]. Prikladom je COMMAND [az sql server ad-admin create --resource-group Chapter07 --server ta76dbserver --object-id "5138f3d7-3ec0-4d89-97f5-40a6f6000528" --display-name "Timmy Anderson"]. VALUE [5138f3d7-3ec0-4d89-97f5-40a6f6000528] je ID AZURE ACCOUNT.
      D. OPTIONALLY sa na AZURE SQL SERVER nastavi, aby pouzival iba ACTIVE DIRECTORY AUTHENTICATION pomocou COMMAND [az sql server ad-only-auth enable]. Prikladom je COMMAND [az sql server ad-only-auth enable --resource-group Chapter07 --name ta76dbserver].
      E. Vytvori sa AZURE SQL SERVER DATABASE pomocou COMMAND [az sql db create]. Prikladom je COMMAND [az sql db create --name MyDatabase --server ta76dbserver --resource-group Chapter07 --compute-model Provisioned --edition Basic].
      F. Pouzije sa CONNECTION STRING, ktory ma SYNTAX [Server=tcp:ta76dbserver.database.windows.net,1433;Initial Catalog=MyDatabase;Encrypt=True;TrustServerCertificate=False;Connection Timeout=30;Authentication="Active Directory Default";].
21. Pre AUTHENTICATION [SQL USER NAME PASSWORD] platia nasledujuce fakty.
   A. Tento typ AUTHENTICATION je mozne pouzit aj z EXTERNAL ENVIRONMENTS, ktore bezia mimo AZURE.
   B. USER NAME a PASSWORD, ktore sa pouziju na AUTHENTICATION su sucastou CONNECTION STRING.
   C. V AZURE SQL SERVER a v AZURE SQL SERVER DATABASE musi byt pridany dany SQL USER.
   D. AUTHENTICATION pre AZURE SQL SERVER ADMIN USER sa vykonava v nasledujucich krokoch.
      A. Vytvori sa AZURE SQL SERVER pomocou COMMAND [az sql server create]. Prikladom je COMMAND [az sql server create --name ta76dbserver --resource-group Chapter07 --location westus --admin-user mydbadmin --admin-password "Qaxwsx123+."]. COMMAND MUSI obsahovat definiciu AZURE SQL SERVER ADMIN USER a AZURE SQL SERVER ADMIN PASSWORD.
      B. Na AZURE SQL SERVER sa vytvori FIREWALL RULE, ktory povoli pristup z LOCAL MACHINE pomocou COMMAND [az sql server firewall-rule create]. Prikladom je COMMAND [az sql server firewall-rule create --name rulemyip --resource-group Chapter07 --server ta76dbserver --start-ip-address 178.40.58.85 --end-ip-address 178.40.58.85].
      C. Vytvori sa AZURE SQL SERVER DATABASE pomocou COMMAND [az sql db create]. Prikladom je COMMAND [az sql db create --name MyDatabase --server ta76dbserver --resource-group Chapter07 --compute-model Provisioned --edition Basic].
      D. Pouzije sa CONNECTION STRING, ktory ma SYNTAX [Server=tcp:ta76dbserver.database.windows.net,1433;Initial Catalog=MyDatabase;Persist Security Info=False;User ID=mydbadmin;Password=Qaxwsx123+.;MultipleActiveResultSets=False;Encrypt=True;TrustServerCertificate=False;Connection Timeout=30;].
22. Pre AUTHENTICATION [SYSTEM ASSIGNED MANAGED IDENTITY] platia nasledujuce fakty.
   A. Tento typ AUTHENTICATION je mozne pouzit IBA z AZURE ENVIRONMENT.
   B. AUTHENTICATION sa typicky pouziva pre PROGRAMS, ktore bezia v AZURE a potrebuju sa pripojit na AZURE SQL SERVER DATABASE.
   C. AUTHENTICATION vyuziva SYSTEM ASSIGNED MANAGED IDENTITY, ktora je spravidla vytovrena spolu s RESOURCE, ktory sa potrebuje na AZURE SQL SERVER DATABASE. V pripade, ze sa na AZURE SQL SERVER DATABASE potrebuje pripojit WEB APPLICATION< tak SYSTEM ASSIGNED MANAGED IDENTITY je asociovany s WEB APPLICATION.
   D. CONNECTION STRING neobsahuje USER NAME ani PASSWORD, takze tento sposob AUTHENTICATION je bezpecnejsi ako pouzitie USER NAMES a PASSWORDS.
   E. AUTHENTICATION pre AZURE SQL SERVER pre WEB APPLICATION sa vykonava v nasledujucich krokoch.
      A. Vytvori sa AZURE SQL SERVER pomocou COMMAND [az sql server create]. Prikladom je COMMAND [az sql server create --name ta76dbserver --resource-group Chapter07 --location westus --admin-user mydbadmin --admin-password "Qaxwsx123+."].
      B. Na AZURE SQL SERVER sa vytvori FIREWALL RULE, ktory povoli pristup pre AZURE SERVICES pomocou COMMAND [az sql server firewall-rule create]. Prikladom je COMMAND [az sql server firewall-rule create --name AllowAzureService --resource-group Chapter07 --server ta76dbserver --start-ip-address 0.0.0.0 --end-ip-address 0.0.0.0].
      C. Vytvori sa APP SERVICE PLAN pomocou COMMAND [az appservice plan create]. Prikladom je COMMAND [az appservice plan create --name MyAppServicePlan --resource-group Chapter07 --location centralus --is-linux --sku FREE].
      D. Vytvori sa WEB APPLICATION pomocou COMMAND [az webapp create]. Prikladom je COMMAND [az webapp create --name MyWebProgram --plan MyAppServicePlan --resource-group Chapter07 --container-image-name docker.io/timmyanderson/using_azure_sql_database:1.0.0].
      !!! E. Pre WEB APPLICATION sa vytvori SYSTEM ASSIGNED MANAGED IDENTITY pomocou COMMAND [az webapp identity assign]. Prikladom je COMMAND [az webapp identity assign --name MyWebProgram --resource-group Chapter07 --identities [system]].
      !!!!! F. Z WEB APPLICATION SYSTEM ASSIGNED MANAGED IDENTITY sa vytvori AZURE SQL SERVER ADMINISTRATOR volanim COMMAND [az sql server ad-admin create]. Prikladom je COMMAND [az sql server ad-admin create --resource-group Chapter07 --server ta76dbserver --object-id "d4ab845e-2b8f-4e1d-9649-b2f7f8584566" --display-name "My System Managed Identity"]. VALUE [d4ab845e-2b8f-4e1d-9649-b2f7f8584566] je PRINCIPAL ID SYSTEM ASSINGNED MANAGED IDENTITY, ktora bola pridelena WEB APPLICATION. VALUE sa ziskava z COMMAND [az webapp identity show --name MyWebProgram --resource-group Chapter07] z PROPERTY [principalId].
      G. OPTIONALLY sa na AZURE SQL SERVER nastavi, aby pouzival iba ACTIVE DIRECTORY AUTHENTICATION pomocou COMMAND [az sql server ad-only-auth enable]. Prikladom je COMMAND [az sql server ad-only-auth enable --resource-group Chapter07 --name ta76dbserver].
      H. Vytvori sa AZURE SQL SERVER DATABASE pomocou COMMAND [az sql db create]. Prikladom je COMMAND [az sql db create --name MyDatabase --server ta76dbserver --resource-group Chapter07 --compute-model Provisioned --edition Basic].
      I. Pouzije sa CONNECTION STRING, ktory ma SYNTAX [Server=tcp:ta76dbserver.database.windows.net,1433;Initial Catalog=MyDatabase;Encrypt=True;TrustServerCertificate=False;Connection Timeout=30;Authentication=Active Directory Managed Identity;].
   F. AUTHENTICATION pre AZURE SQL SERVER pre WEB APPLICATION bez POUZITIA ADMIN RIGHTS sa vykonava v nasledujucich krokoch.
      A. Vytvori sa AZURE SQL SERVER pomocou COMMAND [az sql server create]. Prikladom je COMMAND [az sql server create --name ta76dbserver --resource-group Chapter07 --location westus --admin-user mydbadmin --admin-password "Qaxwsx123+."].
      B. Na AZURE SQL SERVER sa vytvori FIREWALL RULE, ktory povoli pristup pre AZURE SERVICES pomocou COMMAND [az sql server firewall-rule create]. Prikladom je COMMAND [az sql server firewall-rule create --name AllowAzureService --resource-group Chapter07 --server ta76dbserver --start-ip-address 0.0.0.0 --end-ip-address 0.0.0.0].
      C. Vytvori sa APP SERVICE PLAN pomocou COMMAND [az appservice plan create]. Prikladom je COMMAND [az appservice plan create --name MyAppServicePlan --resource-group Chapter07 --location centralus --is-linux --sku FREE].
      D. Vytvori sa WEB APPLICATION pomocou COMMAND [az webapp create]. Prikladom je COMMAND [az webapp create --name MyWebProgram --plan MyAppServicePlan --resource-group Chapter07 --container-image-name docker.io/timmyanderson/using_azure_sql_database:1.0.0].
      !!! E. Pre WEB APPLICATION sa vytvori SYSTEM ASSIGNED MANAGED IDENTITY pomocou COMMAND [az webapp identity assign]. Prikladom je COMMAND [az webapp identity assign --name MyWebProgram --resource-group Chapter07 --identities [system]].
      !!!!! F. Z WEB APPLICATION SYSTEM ASSIGNED MANAGED IDENTITY sa vytvori AZURE SQL SERVER ADMINISTRATOR volanim COMMAND [az sql server ad-admin create]. Prikladom je COMMAND [az sql server ad-admin create --resource-group Chapter07 --server ta76dbserver --object-id "d4ab845e-2b8f-4e1d-9649-b2f7f8584566" --display-name "My System Managed Identity"]. VALUE [d4ab845e-2b8f-4e1d-9649-b2f7f8584566] je PRINCIPAL ID SYSTEM ASSINGNED MANAGED IDENTITY, ktora bola pridelena WEB APPLICATION. VALUE sa ziskava z COMMAND [az webapp identity show --name MyWebProgram --resource-group Chapter07] z PROPERTY [principalId].
      G. OPTIONALLY sa na AZURE SQL SERVER nastavi, aby pouzival iba ACTIVE DIRECTORY AUTHENTICATION pomocou COMMAND [az sql server ad-only-auth enable]. Prikladom je COMMAND [az sql server ad-only-auth enable --resource-group Chapter07 --name ta76dbserver].
      H. Vytvori sa AZURE SQL SERVER DATABASE pomocou COMMAND [az sql db create]. Prikladom je COMMAND [az sql db create --name MyDatabase --server ta76dbserver --resource-group Chapter07 --compute-model Provisioned --edition Basic].
      I. Pouzije sa CONNECTION STRING, ktory ma SYNTAX [Server=tcp:ta76dbserver.database.windows.net,1433;Initial Catalog=MyDatabase;Encrypt=True;TrustServerCertificate=False;Connection Timeout=30;Authentication=Active Directory Managed Identity;].
      !!! J. Teraz CLIENT PROGRAM moze pristupit na WEB APPLICATION, ktora pristupi na AZURE SQL SERVER DATABASE s pouzitim ADMIN ACCOUNT.
      !!!!! K. AZURE USER ACCOUNT sa nastavi ako ADMINISTRATOR ACCOUNT pre AZURE SQL SERVER pomocou pomocou COMMAND [az sql server ad-admin create]. Prikladom je COMMAND [az sql server ad-admin create --resource-group Chapter07 --server ta76dbserver --object-id "5138f3d7-3ec0-4d89-97f5-40a6f6000528" --display-name "Timmy Anderson"]. VALUE [5138f3d7-3ec0-4d89-97f5-40a6f6000528] je ID AZURE ACCOUNT. Tym padom SYSTEM ASSIGNED MANAGED IDENTITY uz NEBUDE ADMIN ACCOUNT.

Prikladom je COMMAND [az sql server firewall-rule create --name rulemyip --resource-group Chapter07 --server ta76dbserver --start-ip-address 178.40.58.85 --end-ip-address 178.40.58.85]. Tento COMMAND je nutny, aby bolo mozne vykonat SQL COMMANDS nad AZURE SQL SERVER DATABASE.
      !!!!! M. Nad AZURE SQL SERVER DATABASE sa vykonaju nasledujuce SQL COMMANDS.
         A. COMMAND [CREATE USER [MyWebProgram] FROM EXTERNAL PROVIDER;] vytvori DATABASE USER v DATABASE. VALUE [MyWebProgram] je NAZOV WEB APPLICATION.
